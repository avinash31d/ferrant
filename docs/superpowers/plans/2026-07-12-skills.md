# Skills Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add secure progressive-disclosure loading of local and GitHub-hosted Codex/Claude skill packages to Ferrant agents.

**Architecture:** A focused `skills` module parses packages, materializes configured sources, and owns safe resource access. `SkillCatalog` exposes two ordinary Ferrant tools and a compact prompt fragment; `AgentBuilder` composes those into the existing agent loop without changing providers.

**Tech Stack:** Rust 2021, Tokio, serde/serde_yaml, thiserror, std::process `git`, existing Ferrant `Tool` and `AgentBuilder` APIs.

## Global Constraints

- Skills use a directory containing `SKILL.md` with YAML frontmatter requiring `name` and `description`.
- Unknown YAML metadata is preserved.
- Sources support recursive local discovery and GitHub repository URL, optional Git reference, and optional subdirectory.
- Skills and resources are loaded as inert content only; Ferrant never executes scripts or installs dependencies.
- Canonical paths must remain under the selected skill root after symlink resolution.
- Instruction and resource reads have configurable byte limits.
- Source loading is atomic: any discovery or parse error prevents a partial catalog.
- Git arguments are passed directly to `Command`, never through a shell.
